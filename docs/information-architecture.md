# Information Architecture

## 1. Purpose

定义第一版 demo 的页面结构、页面职责、组件边界和核心交互入口。

## 2. Scope

覆盖：

- 首页
- 历史记录
- 仪表盘
- 我
- 底部导航
- 页面级交互入口
- 页面内模块职责

不覆盖：

- 登录流
- 设置中心的复杂配置
- 视觉稿级说明

## 3. Global Structure

底部导航包含四个一级页面：

1. 首页
2. 历史记录
3. 仪表盘
4. 我

全局规则：

- 首页是默认入口页
- 四个一级页面都保留底部导航，支持随时切换
- 一级页面之间不通过深层路由传递复杂状态，页面展示以本地存储中的当前状态为准
- 首页负责“开始 / 停止任务”，其他页面不承担任务启动职责
- 历史记录页负责事后修正，避免首页承载补录复杂度
- “我”页负责项目体系维护，不直接承担历史修正和统计解释职责

## 4. Home

### 4.1 Purpose

用于最低摩擦地开始或停止任务。

职责边界：

- 承担所有“开始任务”入口
- 展示当前进行中的任务状态
- 提供项目发现、搜索和排序
- 不承担项目管理细节编辑
- 不承担历史记录修正

### 4.2 Layout

从上到下：

1. 顶部搜索区
2. 当前本人任务区
3. 运行中的 Agent 任务区
4. 项目列表控制区
5. 可开始的具体项目列表
6. 搜索空状态入口
7. 底部导航

### 4.3 Structure Details

#### 4.3.1 顶部搜索区

组件职责：

- 输入具体项目名称关键词
- 在当前未归档项目范围内过滤结果
- 作为“搜索不到时进入创建”的判断入口

#### 4.3.2 当前本人任务区

组件职责：

- 仅显示当前唯一进行中的本人任务
- 展示项目名、项目类型、开始时间、已运行时长
- 提供停止入口

空状态：

- 没有本人任务时显示“当前无本人任务进行中”

#### 4.3.3 运行中的 Agent 任务区

组件职责：

- 展示全部进行中的 Agent 任务
- 每条显示项目名、项目类型、开始时间、已运行时长
- 每条提供独立停止入口

空状态：

- 没有进行中的 Agent 任务时，该区可收起或显示简短占位

#### 4.3.4 项目列表控制区

组件职责：

- 切换项目列表排序模式
- 第一版只包含：
  - 最近使用
  - 最常使用

#### 4.3.5 具体项目列表

组件职责：

- 展示可直接点击开始的未归档具体项目
- 每项显示：
  - 具体项目名
  - 项目类型
  - 执行主体
- 点击本人项目时执行“自动结束上一个本人任务，再开始新的”
- 点击 Agent 项目时直接新增并行任务

#### 4.3.6 搜索空状态入口

组件职责：

- 当搜索结果为空时，明确告知当前无匹配项目
- 提供首页内的轻量快速创建入口
- 快速创建只收集最小必要信息：
  - 具体项目名
  - 项目类型
  - 执行主体
- 完整的项目管理仍归“我”页负责

### 4.4 Key Interactions

- 搜索具体项目
- 切换排序模式：
  - 最近使用
  - 最常使用
- 点击具体项目立即开始
- 停止当前本人任务
- 停止某个 Agent 任务
- 搜索不到时在首页快速创建项目

### 4.5 Page Rules

- 首页项目列表不按项目类型分组展开，保持扁平低摩擦
- 归档项目不出现在首页
- 首页不提供项目编辑、归档、改类型、改执行主体入口
- 首页允许轻量快速创建具体项目，但不承担完整项目管理
- 首页可展示运行中状态，但不负责修改历史时间

## 5. History

### 5.1 Purpose

查看和修正历史记录。

职责边界：

- 承担历史查询、排序、展开和修正
- 承担备注补录
- 不承担新建项目和编辑项目定义
- 不承担统计分析解释

### 5.2 Layout

从上到下：

1. 页面标题
2. 排序控件区
3. 历史记录列表
4. 单条记录展开编辑区
5. 底部导航

### 5.3 Structure Details

#### 5.3.1 排序控件区

组件职责：

- 切换历史记录展示顺序
- 第一版只包含：
  - 时间倒序
  - 时间正序
  - 耗时排序

#### 5.3.2 历史记录列表

组件职责：

- 展示所有历史记录
- 默认以“具体项目名”为主标题
- 每条记录默认只占一行或一卡
- 点击单条记录后在原位置展开编辑内容

#### 5.3.3 单条记录展开编辑区

组件职责：

- 编辑开始时间
- 编辑结束时间
- 编辑备注
- 保存后回写记录并刷新排序与统计关联结果

### 5.4 Record Card

收起态显示：

- 具体项目名
- 执行主体
- 开始时间
- 结束时间
- 耗时
- 项目类型

展开态显示：

- 上述信息
- 开始时间编辑
- 结束时间编辑
- 备注编辑
- 保存按钮

### 5.5 Key Interactions

- 切换历史排序
- 展开 / 收起单条记录
- 编辑开始时间
- 编辑结束时间
- 编辑备注
- 保存修改

### 5.6 Page Rules

- 历史记录页不承担“开始任务”
- 历史记录编辑是原地展开，不跳转详情页
- 历史记录展示名称和项目类型随项目定义变更同步更新
- 结束时间不能早于开始时间，校验失败时应在本页内提示

## 6. Dashboard

### 6.1 Purpose

提供基础时间统计，不承担任务启动职责。

职责边界：

- 承担统计展示
- 提供周期切换
- 不承担项目管理
- 不承担历史记录编辑
- 不承担任务开始 / 停止

### 6.2 Layout

从上到下：

1. 页面标题
2. 周期切换区
3. 核心统计卡片区
4. 项目类型分布区
5. 最近趋势区
6. 底部导航

### 6.3 Structure Details

#### 6.3.1 周期切换区

组件职责：

- 在日 / 周 / 月之间切换统计视角
- 当前选择影响下方卡片、分布和趋势

#### 6.3.2 核心统计卡片区

组件职责：

- 展示当前周期下的核心统计结果
- 区分本人时间和 Agent 时间，避免口径混淆

#### 6.3.3 项目类型分布区

组件职责：

- 展示时间在不同项目类型上的分布
- 主要服务于结构性回顾，不承担明细查账

#### 6.3.4 最近趋势区

组件职责：

- 展示最近一段时间的变化趋势
- 服务于用户识别投入变化，而不是精细分析

### 6.4 Metrics

第一版至少包括：

- 当前周期本人投入
- 当前周期 Agent 耗时
- 当前周期项目类型分布
- 当前周期趋势

补充口径：

- 周期切换到“日”时，优先展示当日本人 / Agent 耗时
- 周期切换到“周 / 月”时，展示对应周期本人 / Agent 耗时
- 项目类型分布默认以当前周期内已结束记录为基础

### 6.5 Key Interactions

- 切换统计周期
- 浏览统计卡片
- 浏览项目类型分布
- 浏览最近趋势

### 6.6 Page Rules

- 仪表盘不提供任务开始入口
- 仪表盘不允许直接编辑统计数据
- Agent 时间单列展示，不混入本人投入

## 7. Me

### 7.1 Purpose

维护项目类型和具体项目。

职责边界：

- 承担项目体系维护
- 提供项目类型和具体项目的创建、编辑、归档、删除入口
- 不承担任务记录编辑
- 不承担统计解释

### 7.2 Layout

分两块：

1. 页面标题与说明区
2. 项目类型管理区
3. 具体项目管理区
4. 底部导航

### 7.3 Structure Details

#### 7.3.1 页面标题与说明区

组件职责：

- 说明本页用于维护项目总表
- 为“从首页搜索不到结果过来”的用户提供明确方向

#### 7.3.2 项目类型管理区

组件职责：

- 展示现有项目类型
- 提供新建、编辑、删除入口
- 在删除受限时给出原因提示

每个类型项至少显示：

- 类型名称
- 关联具体项目数量
- 编辑入口
- 删除入口

#### 7.3.3 具体项目管理区

组件职责：

- 展示全部具体项目
- 支持按未归档 / 已归档区分查看
- 提供新建、编辑、归档入口
- 让用户清楚看到项目与类型、执行主体的绑定关系

### 7.4 Type Management

能力：

- 新建项目类型
- 编辑项目类型
- 删除空项目类型

删除规则提示：

- 如果类型下仍有未归档或已存在的具体项目，不允许直接删除
- 页面需要明确提示“需先处理该类型下具体项目”

### 7.5 Project Management

能力：

- 新建具体项目
- 编辑具体项目
- 指定项目类型
- 指定执行主体
- 归档具体项目

补充要求：

- 新建具体项目时必须先选项目类型
- 新建具体项目时必须绑定执行主体
- 编辑具体项目时允许更换项目类型
- 已归档项目不出现在首页，但仍可在本页查看和管理其归档状态信息

### 7.6 Key Interactions

- 新建项目类型
- 编辑项目类型
- 删除项目类型
- 新建具体项目
- 编辑具体项目
- 归档具体项目
- 查看已归档项目

### 7.7 Page Rules

- “我”页承担从零维护项目体系的职责，是首页空搜索结果的承接页
- 删除项目类型必须受“仍存在具体项目”约束
- 具体项目的执行主体是项目定义的一部分，不在首页临时切换
- 本页可以编辑项目定义，但不直接修改历史记录内容

## 8. Navigation Rules

- 首页是默认入口
- 从首页搜索不到项目时，可进入“我”页创建
- 任何页面都通过底部导航返回其他一级页面
- 从“我”页完成新建具体项目后，应能回到首页立即开始使用
- 历史记录和仪表盘不承担跨页创建流程的主入口

## 9. Product Alignment Check

已对照 `docs/product-spec.md` 校验，当前信息架构与产品规则总体一致，结论如下：

- 首页仍然是唯一任务启动主入口，符合规格
- 历史记录页承担补录和修正，符合规格
- 仪表盘只做统计展示，不承接首页功能，符合规格
- “我”页承担项目类型和具体项目维护，符合规格
- 删除项目类型的前置限制已补到页面职责中，避免和产品规则脱节

## 10. Risks

当前非阻塞风险：

- 产品规格要求“搜索不到时允许快速创建新具体项目”，但当前页面层定义为“跳转到我页创建”，属于保守实现。两者不冲突，但“快速创建”的速度预期仍可进一步明确。
- 仪表盘指标定义里写了“今日本人投入 / 本周本人投入 / 本月本人投入 / 累计 Agent 耗时”，与“日 / 周 / 月周期切换”并存时，最终卡片文案需要在实现前统一，避免出现固定口径和切换口径并列造成理解歧义。
