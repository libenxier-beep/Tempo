# Information Architecture

## 1. Purpose

定义第一版 demo 的页面结构、页面职责和核心交互入口。

## 2. Scope

覆盖：

- 首页
- 历史记录
- 仪表盘
- 我
- 底部导航

不覆盖：

- 登录流
- 设置中心的复杂配置

## 3. Global Structure

底部导航包含四个一级页面：

1. 首页
2. 历史记录
3. 仪表盘
4. 我

## 4. Home

### 4.1 Purpose

用于最低摩擦地开始或停止任务。

### 4.2 Layout

从上到下：

1. 搜索框
2. 当前本人任务大卡片
3. 运行中的 Agent 任务列表
4. 具体项目列表
5. 底部导航

### 4.3 Key Interactions

- 搜索具体项目
- 切换排序模式：
  - 最近使用
  - 最常使用
- 点击具体项目立即开始
- 停止当前本人任务
- 停止某个 Agent 任务
- 搜索不到时跳转到“我”页创建项目

## 5. History

### 5.1 Purpose

查看和修正历史记录。

### 5.2 Layout

从上到下：

1. 页面标题
2. 排序控件
3. 历史记录列表
4. 底部导航

### 5.3 Record Card

收起态显示：

- 具体项目名
- 执行主体
- 开始时间
- 结束时间
- 耗时
- 项目类型

展开态显示：

- 上述信息
- 开始时间编辑
- 结束时间编辑
- 备注编辑
- 保存按钮

## 6. Dashboard

### 6.1 Purpose

提供基础时间统计，不承担任务启动职责。

### 6.2 Layout

从上到下：

1. 页面标题
2. 统计卡片区
3. 项目类型分布
4. 最近趋势
5. 底部导航

### 6.3 Metrics

第一版至少包括：

- 今日本人投入
- 本周本人投入
- 本月本人投入
- 累计 Agent 耗时

## 7. Me

### 7.1 Purpose

维护项目类型和具体项目。

### 7.2 Layout

分两块：

1. 项目类型管理
2. 具体项目管理

### 7.3 Type Management

能力：

- 新建项目类型
- 编辑项目类型
- 删除空项目类型

### 7.4 Project Management

能力：

- 新建具体项目
- 编辑具体项目
- 指定项目类型
- 指定执行主体
- 归档具体项目

## 8. Navigation Rules

- 首页是默认入口
- 从首页搜索不到项目时，可进入“我”页创建
- 任何页面都通过底部导航返回其他一级页面
