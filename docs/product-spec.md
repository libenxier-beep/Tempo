# Product Spec

## 1. Purpose

定义“柳比歇夫时间记录”第一版 demo 的产品目标、边界和核心规则，保证后续实现不再靠聊天记录回忆。

## 2. Scope

覆盖范围：

- 手机端优先的最小闭环产品
- 本地存储
- 首页、历史记录、仪表盘、我
- 项目类型、具体项目、任务记录
- 本人任务和 Agent 任务的行为差异

不覆盖：

- 登录
- 云同步
- 推送提醒
- 桌面组件
- 多人协作

## 3. Product Goal

做一个用户自己能在生活中长期使用的时间记录产品，而不是单纯展示用原型。

第一版目标是跑通最小闭环：

1. 先维护项目体系
2. 首页快速开始任务
3. 任务结束后形成记录
4. 在历史里修正和补充备注
5. 在仪表盘看基础统计

## 4. Core Concepts

### 4.1 项目类型

一级字段，例如：

- 生活
- 工作
- 学习

### 4.2 具体项目

实际可被点击开始的对象，必须挂在某个项目类型下面，例如：

- 生活 -> 散步
- 生活 -> 陪家人
- 工作 -> 产品设计
- 工作 -> 跑训练

### 4.3 执行主体

固定绑定在具体项目上，第一版取值只有：

- 本人
- Agent

具体项目创建时就确定执行主体，第一版不支持后续按单次记录临时更改。

### 4.4 任务记录

一次具体项目的开始到结束，形成一条历史记录。

## 5. User Flow

### 5.1 首页启动任务

1. 进入首页
2. 搜索或直接点选具体项目
3. 立即开始任务

### 5.2 本人任务规则

- 点击本人项目后立即开始计时
- 如果已有本人任务正在进行，系统自动结束上一个本人任务，再开始新的
- 本人任务同一时刻只允许一个处于进行中

### 5.3 Agent 任务规则

- 点击 Agent 项目后立即开始
- Agent 任务可并行存在多个
- Agent 任务不计入本人时间统计
- Agent 任务通过手动点击停止结束

### 5.4 结束和补录

- 本人任务和 Agent 任务都通过手动停止结束
- 如果忘记停止，允许在历史记录中修改开始时间和结束时间
- 备注不拦截开始流程，统一在历史记录中补充

## 6. Page Requirements

### 6.1 首页

页面结构：

- 顶部搜索框
- 中央大卡片：当前本人任务
- 下方小列表：运行中的 Agent 任务
- 再下方：可点击开始的具体项目列表
- 底部导航

首页规则：

- 直接显示具体项目，不先显示项目类型层级
- 支持搜索具体项目
- 搜索不到时允许快速创建新具体项目
- 项目列表支持两种排序模式：
  - 最近使用
  - 最常使用
- 每个具体项目显示：
  - 具体项目名
  - 项目类型
  - 执行主体

### 6.2 历史记录

历史记录中的每一条以“具体项目名”为主体显示。

默认显示：

- 具体项目名
- 开始时间
- 结束时间
- 自动计算的耗时
- 执行主体
- 项目类型

交互规则：

- 支持三种排序：
  - 时间倒序
  - 时间正序
  - 花费时间长短
- 点开一条记录后原地展开
- 展开后允许编辑：
  - 开始时间
  - 结束时间
  - 普通文字备注

### 6.3 仪表盘

第一版只做基础统计，不承担首页功能。

统计周期：

- 日
- 周
- 月

至少包含：

- 本人时间统计
- Agent 时间统计
- 项目类型分布
- 最近趋势

### 6.4 我

用于维护项目总表。

必须支持：

- 新建项目类型
- 编辑项目类型
- 删除项目类型
- 新建具体项目
- 编辑具体项目
- 归档具体项目

## 7. Management Rules

### 7.1 新建具体项目

必须同时满足：

- 先选择一个项目类型
- 绑定固定执行主体

否则不允许创建。

### 7.2 删除具体项目

第一版按“归档”处理：

- 历史记录保留
- 首页不再出现
- 不能再用它开始新任务

### 7.3 删除项目类型

不允许直接删除带有具体项目的项目类型。

必须先处理掉其下所有具体项目，才能删除项目类型。

### 7.4 编辑项目归属

允许把具体项目移动到另一个项目类型下面。

一旦移动：

- 该具体项目的历史记录显示的项目类型也同步更新

### 7.5 编辑名称

允许编辑：

- 项目类型名称
- 具体项目名称

编辑后，历史记录里的显示名称同步更新。

## 8. Storage Strategy

第一版仅使用本地存储。

## 9. Success Criteria

第一版完成的判断标准：

1. 用户能在手机端快速开始任务
2. 本人任务切换摩擦足够低
3. Agent 任务能并行记录且不污染本人统计
4. 忘记停止时可以在历史中修正
5. 项目总表能维护，不会越用越脏
